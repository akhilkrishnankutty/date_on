## 2024-05-05 - [IDOR] Profile Image Exposure Before Match Time Limit
**Vulnerability:** The API returned the unblurred `profilePictureUrl` immediately upon a match, relying entirely on the frontend application to hide or blur the image until 5 days had elapsed since the `matchTime`.
**Learning:** This architectural flaw exists when developers trust client-side code for authorization or data presentation logic. Attackers can intercept the JSON response directly from the API and extract the clear text URL for the profile picture regardless of UI-level hiding/blurring mechanisms.
**Prevention:** Never trust the client application to mask sensitive data or enforce business rules (like time-based content unlocking). Always evaluate authorization and state requirements on the backend and explicitly exclude restricted data from API responses.
